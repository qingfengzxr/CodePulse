# 图表语义审查记录

## 背景

本文记录当前 Web 图表实现的专项 review，目标不是评价视觉效果，而是确认“图表名称、文案、交互、聚合口径”是否和真实数据语义一致。

`01-views.md` 负责回答“应该用什么图表达什么问题”，本文负责记录“当前实现有没有违背这些设计语义”。

## 高优先级问题

### 1. K 线图使用了并不存在的 OHLC 语义

当前实现问题：

- 当前 `ModuleCandlestickChart` 把模块 `LOC` 演化包装成 OHLC K 线
- `open` 使用上一个采样点 `LOC`
- `close` 使用当前采样点 `LOC`
- `high` / `low` 则用 `open + added`、`open - deleted` 与 `close` 推导

语义风险：

- `added` / `deleted` 是时间桶内累计变更量，不是桶内真实路径
- 当前实现没有观测到时间桶内的真实最高点与最低点
- 因此 `high/low` 不是“真实极值”，只是推导边界
- 一旦用标准 K 线视觉编码，用户会自然把它理解成真实波动区间，这会夸大当前数据能够表达的含义

结论：

- 当前 K 线图不应继续以标准 OHLC 语义对外表达
- 如果保留，需要明确标注它只是基于 `LOC + added/deleted` 推导出的合成区间，而不是真实 intra-period high/low

### 2. 普通堆叠面积图被描述成“占比变化”

当前实现问题：

- `ModuleStackedAreaChart` 使用的是绝对 `LOC` 堆叠
- 但图表文案和详情页卡片描述强调“结构占比”“份额变化”“结构迁移”

语义风险：

- 普通堆叠面积图会同时表达“总量变化 + 模块绝对规模变化”
- 仓库整体变大或变小时，视觉变化会首先被总量主导
- 用户很容易把“绝对值变化”误读为“份额变化”

结论：

- 普通堆叠面积图应被描述为“结构构成 + 绝对体量变化”
- 真正表达份额变化的图只能是 100% 归一化堆叠面积图

## 中优先级问题

### 3. Top N 筛选默认按最新时点排序，会系统性扭曲历史叙事

当前实现问题：

- 堆叠面积图、100% 堆叠面积图、Bump Chart 默认都先按最新时点值排序
- 然后只保留最新时点前 N 个模块，其余模块折叠进 `Others` 或直接不展示

语义风险：

- 某些模块可能在历史中前期非常重要，但在最新时点已经缩小
- 当前策略会把这些模块从整个历史视图里弱化掉
- 这会导致“结构迁移”“谁在上升、谁在掉队”被当前终局状态主导，而不是由完整历史决定

结论：

- 当前实现更适合表达“今天最重要的模块，其历史是怎样的”
- 不适合直接表述成“全周期里最值得看的结构迁移/地位变化”
- 后续若要修正，至少需要重新定义 Top N 口径，例如按累计面积、峰值、时间窗口内最大值或用户自选

### 4. 趋势图里的“隐藏”实际上只是弱化

当前实现问题：

- 模块趋势图文案显示“其余模块已隐藏”
- 但实现仍然会把所有模块 series 渲染到图上，只是降低透明度和线宽

语义风险：

- 用户会以为图里只剩自己选中的模块
- 实际上未选模块仍然参与视觉编码，也可能影响 tooltip、legend 和整体噪音水平

结论：

- 如果继续保留当前实现，文案应改成“弱化”或“降噪显示”
- 如果希望文案成立，就需要真的从图表 series 中移除未选模块

### 5. 离散时间桶指标被画成平滑连续曲线

当前实现问题：

- 趋势图、堆叠面积图、100% 堆叠面积图、Bump Chart 等当前默认都使用 `smooth: true`
- 尤其 `added/deleted/churn` 这些指标本质上是离散时间桶聚合值

语义风险：

- 平滑曲线会凭空创造时间桶之间的中间走势
- 用户可能把曲线弯折理解成连续变化趋势或变化速度
- 这对 `added/deleted/churn` 这类离散 bucket 指标尤其不成立

结论：

- `loc` 是否平滑仍可讨论
- `added/deleted/churn` 不应默认使用平滑曲线

### 6. 当前排行里的“Top 合计”容易被误读为总量

当前实现问题：

- 排行图通过接口只请求 Top N
- 页面上的 `Top 合计` 也是对返回的 Top N 求和

语义风险：

- 用户可能把这个数读成“当前时间点总量”
- 但它只是被截断后的部分和，不能直接与总 `LOC` 或全量 distribution 对比

结论：

- 文案需要明确写成 `Top 8 合计` / `Top 16 合计`
- 或者补出“占全量比例”，否则它会被误解成全局总数

### 7. 详情页概览里的“新增 / 删除 / Churn”没有说明只针对最新采样点

当前实现问题：

- 当前概览卡片展示的 `新增 / 删除 / Churn`
- 实际来源是 `snapshot=latest` 的 distribution 总和

语义风险：

- 用户容易把这个值看成整个分析周期累计新增、累计删除、累计 churn
- 但它只对应最后一个采样桶

结论：

- 概览文案必须明确写成“最新采样新增 / 删除 / Churn”
- 若要展示累计值，需要单独定义累计口径和对应接口

### 8. 排行图卡片 badge 与图内指标切换脱节

当前实现问题：

- 排行卡片 badge 使用的是基于 `loc` 的 `Top 模块`
- 但图内支持切换 `loc / added / deleted / churn`

语义风险：

- 当用户切到 `added/deleted/churn` 时，badge 仍显示 `loc` 第一名
- 这会让 badge 与当前图内排序语义不一致

结论：

- badge 必须随当前排行指标联动
- 或直接移除这个 badge，避免制造一个额外但不准确的“结论提示”

## 当前可接受但需要收口的点

### 9. 100% 堆叠面积图本身语义成立，但会继承 Top N 筛选偏差

当前情况：

- `ModuleShareStackedAreaChart` 已经做了 100% 归一化
- 这张图作为“份额变化图”在图形选择上是成立的

剩余问题：

- 它仍然继承了“按最新时点值截断 Top N”的默认筛选逻辑
- 所以份额迁移叙事仍会被终局排序偏置

结论：

- 图本身比普通堆叠面积图更符合设计目标
- 但仍需和 Top N 口径一起修

### 10. Bump Chart 本身适合看地位变化，但当前默认过滤不等于“最值得看的排名变化”

当前情况：

- Bump Chart 作为“排名变化图”是合理的
- `loc` 与 `churn` 作为两个主要维度也基本成立

剩余问题：

- 当前图里展示的是“最新时点 Top N 模块的历史排名”
- 不是“整个周期里最值得关注的排名轨迹”

结论：

- 图表类型没有问题
- 问题在于默认筛选与卡片文案说得太满

## 修正建议优先级

建议后续按以下顺序修正：

### P0

- 重新定义或下线当前 K 线图的 OHLC 语义
- 修正文案，区分普通堆叠面积图与 100% 堆叠面积图
- 明确“最新采样新增 / 删除 / Churn”的口径

### P1

- 调整 Top N 的默认筛选逻辑，至少不要一律按最新时点值截断
- 修正趋势图“隐藏”与“弱化”的表述和实现
- 移除或重写对 `added/deleted/churn` 的平滑曲线默认配置

### P2

- 排行图补充 Top N 占比或明确局部求和语义
- 让 badge、summary、footer 与当前图内指标完全联动

## 当前结论

在设计层面，图表类型选型大体方向没有错，但当前实现里已经出现了“图形正确、文案过度”“交互存在、语义没闭合”“筛选策略方便实现但会扭曲历史叙事”这三类问题。

后续所有图表迭代，需要明确遵守以下约束：

- 不允许把推导边界说成真实观测值
- 不允许把绝对值图说成占比图
- 不允许把当前时点 Top N 说成全周期代表性 Top N
- 不允许把局部和说成总量
- 不允许把弱化说成隐藏
