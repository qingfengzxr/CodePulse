# 1. 基于真实 Diff 还是基于 LOC 差值推导

## 背景

当前 `code-dance` 已经能够对 Rust 仓库按周采样，并计算 crate 级时间序列。

目前系统中的指标口径如下：

- `loc`: 基于快照统计的模块总代码行数
- `added/deleted/churn`: 基于相邻采样点 `loc` 差值推导得到

其中第二类指标并不是真正的 Git diff 统计，只是净变化近似值。

## 当前实现方式

假设某个模块在两个相邻采样点上的 `loc` 分别为：

- 时间点 A: `loc = 100`
- 时间点 B: `loc = 120`

当前实现会推导为：

- `added = 20`
- `deleted = 0`
- `churn = 20`

如果后一个时间点变为：

- 时间点 C: `loc = 90`

当前实现会推导为：

- `added = 0`
- `deleted = 30`
- `churn = 30`

这本质上是在表达“净变化”，而不是实际发生了多少增删。

## 问题

这种做法有一个核心缺陷：它无法区分“净增长”和“真实开发活动”。

例如：

- 真实情况可能是新增 50 行、删除 30 行
- 最终 `loc` 净增长只有 20 行

但当前系统只能看到：

- `added = 20`
- `deleted = 0`
- `churn = 20`

这样会丢失大量重要信息。

## 候选方案

### 方案 A：继续使用 LOC 差值推导

优点：

- 实现简单
- 计算成本低
- 不需要额外读取 diff 信息

缺点：

- `added/deleted/churn` 语义不准确
- 无法区分净增长与大规模重构
- 不适合做工程活动分析
- 后续很多图表会产生误导

### 方案 B：改为基于 `git diff --numstat` 的真实增删统计

做法：

- 对每一对相邻采样点 `(A, B)` 运行 `git diff --numstat A B`
- 得到每个文件的新增/删除行数
- 再将文件级增删按模块归类并汇总

优点：

- `added`、`deleted`、`churn` 语义正确
- 能区分增长、清理、重构、迁移
- 更适合做柱状图、热力图、活跃度分析
- 与 Git 原始事实一致，更容易解释和验证

缺点：

- 实现复杂度更高
- 计算开销会增加
- 需要明确文件归属策略
- 需要处理重命名、二进制文件等边界情况

## 讨论结论

结论是采用 **方案 B**。

也就是说，后续指标定义调整为：

- `loc`: 继续基于快照统计
- `added`: 基于相邻采样点之间真实 Git diff 的新增行数
- `deleted`: 基于相邻采样点之间真实 Git diff 的删除行数
- `churn`: `added + deleted`

## 选择方案 B 的原因

### 1. 指标语义正确

如果系统继续保留 `added/deleted/churn` 这些命名，就应该保证这些字段表达的是“真实增删”，而不是净变化近似值。

### 2. 更适合分析工程活动

真实 diff 能识别：

- 大规模重构
- 目录迁移
- 代码清理
- 一次性导入
- 模块活跃度变化

这些在净 `loc` 曲线中往往不明显。

### 3. 更适合图表展示

后续这些展示都应该依赖真实 diff：

- `added/deleted` 柱状图
- churn 热力图
- 活跃模块排行
- 某时间窗口内改动最剧烈模块

如果继续使用净变化近似值，这些图表会产生误导。

### 4. 更容易和 Git 结果对齐

真实 diff 统计可以直接与 Git 命令交叉验证，例如：

- `git diff --numstat`
- `git log --numstat`

这让系统结果更可信，也更方便排查误差来源。

## 实施方向

下一步实现策略：

1. 继续保留当前的 `loc` 快照分析逻辑
2. 对相邻采样点 `(A, B)` 读取 `git diff --numstat A B`
3. 将文件级增删按模块归类
4. 汇总得到模块级 `added/deleted/churn`
5. 用真实 diff 结果替换当前基于 LOC 差值推导的逻辑

## 风险点

### 1. 文件归属问题

当文件在两个采样点之间移动模块、重命名或重新组织时，需要明确归属规则。

### 2. 重命名与迁移

需要评估是否启用 rename detection，以及在模块迁移时如何归并解释。

### 3. 性能

真实 diff 统计会增加分析成本，需要结合采样策略和缓存机制控制耗时。

## 当前决定

本次讨论后的正式决定：

- 将 `added/deleted/churn` 从“LOC 差值推导”升级为“基于 `git diff --numstat` 的真实统计”
- `loc` 保持快照统计口径不变
- 后续实现与图表口径统一按这一决定推进
