# 2. 模块级实现规划总控

本文件不再承载所有实现细节，而是作为 `code-dance` 的总控规划文档使用：

- 维护当前阶段目标
- 维护模块拆分与依赖关系
- 汇总每个模块的进度状态

当前阶段不再是“把主链路跑通”，而是“把现有能力整理成可持续扩展的模块化版本”。

优先目标：

- [ ] 完善 provider 链与手工模块配置

- [ ] 补齐更丰富的查询与展示能力
- [ ] 建立 Web 多语言与明暗模式基础能力
- [ ] 为后续 TUI 和更多语言接入打稳接口
- [ ] 提升测试覆盖与大仓库分析体验

## 模块拆分

当前按以下模块管理实现规划：

1. 共享核心：`domain` / `contracts` / `config`
2. Git 与采样：`packages/git`
3. 分析器与 provider：`packages/analyzer`
4. 存储与指标查询：`packages/storage`
5. API 服务：`apps/api`
6. Web 工作台：`apps/web`
7. TUI 与质量保障：`apps/tui`、E2E、关键验证链路

