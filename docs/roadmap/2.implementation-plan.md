# 2. 实现规划

本文件用于把 `code-dance` 当前实现状态和后续工作拆成更可执行的阶段计划。它是对 [初始实现路线图](/home/zxr/work/github/code-dance/docs/roadmap/1.initial-plan.md) 的细化更新。

## 当前状态摘要

相比最初的规划，当前代码已经超出“Rust 优先 MVP”的范围，现状如下：

- [x] 支持本地 Git 仓库注册
- [x] 支持 Rust 与 Node/Web 两类仓库分析
- [x] 支持 `weekly`、`daily`、`per-commit` 采样
- [x] 支持统一 analyzer 聚合入口
- [x] 支持模块级 `loc`、`added`、`deleted`、`churn`
- [x] 支持 SQLite 持久化
- [x] 支持本地 API 查询
- [x] 支持 Web 页面发起分析、查看进度和查看图表
- [x] 已抽出共享的并发 LOC 统计器

## 当前阶段目标

下一阶段不再是“把主链路做出来”，而是“补足可用性和扩展性缺口”。

优先目标：

- [ ] 完善 provider 链与手工模块配置
- [ ] 完善重复分析检测与缓存策略
- [ ] 补齐更丰富的查询与展示能力
- [ ] 为后续 TUI 和更多语言接入打稳接口
- [ ] 提升测试覆盖与大仓库分析体验
